## 技术推演路线

Requests 静态抓取（入门工程化）
    → Selenium 动态渲染（处理复杂交互）
    → API 接口分析（高效采集思维）
    → Scrapy + Selenium 中间件（框架化 + 可扩展架构）
    → 下一步：Scrapy-Redis 分布式 + JS 逆向

### 项目对比：采集方式的演进

| 维度       | 豆瓣电影 Top250        | 网易新闻（Selenium）        | 网易新闻（API 分析）         | 网易新闻（Scrapy+Selenium）     |
| :--------- | :--------------------- | :-------------------------- | :--------------------------- | :------------------------------ |
| 采集方式   | Requests 直接请求 HTML | Selenium 模拟浏览器渲染     | Requests 直接请求 API 接口   | Scrapy框架 + Selenium中间件     |
| 加载方式   | URL 分页（静态翻页）   | 滚动 + 点击"加载更多"       | API 参数翻页                 | 中间件自动加载动态内容          |
| 数据格式   | HTML → Xpath 解析      | HTML → Xpath 解析           | JSONP → 正则剥离 → JSON 解析 | HTML → Xpath 解析               |
| 核心难点   | 模块化拆分、数据去重   | 动态内容触发、按钮定位      | JSONP 解析、正则提取         | 中间件集成Selenium、框架化调度  |
| 存储模块   | 字段写死，与业务耦合   | 通用模块，零改动复用        | 通用模块，零改动复用         | 通用模块，Pipeline自动触发      |
| 反爬策略   | UA 轮换 + 请求延时     | UA 伪装 + 无头 + 反检测参数 | UA 轮换 + Cookie + Referer   | UA随机轮换 + 无头 + 反检测参数  |
| 效率       | 较高                   | 较低（浏览器渲染）          | 最高（直接请求 JSON）        | 中等（框架调度 + 批量入库优化） |
| 工程化程度 | 初识模块化，文档规范   | 通用存储组件落地            | 通用存储组件验证             | 框架标准化 + 中间件可扩展       |
