## 技术推演路线

静态页面抓取（Requests + Xpath）
    → 动态渲染抓取（Selenium 无头浏览器）
    → API 接口分析（JSONP 解析 + 正则）
    → 框架与分布式（Scrapy + Redis）
    → JS 逆向

### 项目对比：从静态到动态再到高效采集


| 维度     | 豆瓣电影 Top250        | 网易新闻（Selenium）        | 网易新闻（API 分析）         |
| :------- | :--------------------- | :-------------------------- | :--------------------------- |
| 采集方式 | Requests 直接请求 HTML | Selenium 模拟浏览器渲染     | Requests 直接请求 API 接口   |
| 加载方式 | URL 分页（静态翻页）   | 滚动 + 点击"加载更多"       | API 参数翻页                 |
| 数据格式 | HTML → Xpath 解析      | HTML → Xpath 解析           | JSONP → 正则剥离 → JSON 解析 |
| 核心难点 | 模块化拆分、数据去重   | 动态内容触发、按钮定位      | JSONP 解析、正则提取         |
| 存储模块 | 字段写死，与业务耦合   | 通用模块，零改动复用        | 通用模块，零改动复用         |
| 反爬策略 | UA 轮换 + 请求延时     | UA 伪装 + 无头 + 反检测参数 | UA 轮换 + Cookie + Referer   |
| 效率     | 较高                   | 较低（浏览器渲染）          | 最高（直接请求 JSON）        |
